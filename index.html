UniProt Load Data and Logs

UniProt Load (uniprotload) Data and Logs

Data

  • Input Files
    • uniprotmus.dat - from MouseBLAST
    • ec2go - from Gene Ontology, EC-to-GO
    • interpro2go - from Gene Ontology, InterPro-to-GO
    • spkw2go - from Gene Ontology, SwissProt keyword-to-GO

  • Output Files generated from the UniProt Parser
    • uniprot_acc.assoc.txt - all UniProt accession ids, EntrezGene, Ensembl, EC, PDB, InterPro, SwissProt key words
    • uniprot_sp.assoc.txt - SwissProt accession ids
    • uniprot_tr.assoc.txt - TrEMBL accession ids

  • Output Files generated from the MGI Parser (J:53168)
    • mgi_acc.assoc.txt - all MGI, EntrezGene

  • Output Files generated by the Bucketizer: from the UniProt/MGI parser files
    • Note: left-hand side is MGI ID, right-hand side is UniProt ID
    • Note: all of the N buckets contain 2 formats in their files.
    • mgi_uniprot.1_1.txt
    • mgi_uniprot.1_N.txt
    • mgi_uniprot.0_1.txt
    • mgi_uniprot.1_0.txt
    • mgi_uniprot.1_0.proteincoding.txt
    • mgi_uniprot.N_1.txt
    • mgi_uniprot.N_N.txt

  • Output Files generated by Bucketizer: saved files from the last bucketizer
    • Note: left-hand side is MGI ID, right-hand side is UniProt ID
    • mgi_uniprot.1_1.txt.save
    • mgi_uniprot.1_N.txt.save
    • mgi_uniprot.0_1.txt.save
    • mgi_uniprot.1_0.txt.save
    • mgi_uniprot.N_1.txt.save
    • mgi_uniprot.N_N.txt.save

  • Output Files generated by "diffing" the old/saved UniProt bucketes with the new UniProt buckets
    • Note: left-hand side is MGI ID, right-hand side is UniProt ID
    • mgi_uniprot.1_1.txt.lose
    • mgi_uniprot.1_1.txt.gain
    • mgi_uniprot.1_N.txt.lose
    • mgi_uniprot.1_N.txt.gain
    • mgi_uniprot.0_1.txt.lose
    • mgi_uniprot.0_1.txt.gain
    • mgi_uniprot.1_0.txt.lose
    • mgi_uniprot.1_0.txt.gain
    • mgi_uniprot.N_1.txt.lose
    • mgi_uniprot.N_1.txt.gain
    • mgi_uniprot.N_N.txt.lose
    • mgi_uniprot.N_N.txt.gain

  • Output Files generated by comparing the existing SwissProt associations with the new UniProt Buckets
    • left-hand side is new/UniProt, right-hand side is old/SwissProt
    • mgi_swissprot_load.txt - existing SwissProt associations in the database
    • mgi_swissprot.0_1.txt - lost
    • mgi_swissprot.1_0.txt - gained
    • mgi_swissprot.1_1.txt - stayed the same
    • mgi_swissprot.1_N.txt
    • mgi_swissprot.N_1.txt
    • mgi_swissprot.N_N.txt

  • Output Files generated by the Bucketizer as input into the Association loader (assocload)
    • mgi_uniprot_load.txt - UniProt Load file, J:53168, 1_1 and 1_N

  • Output Files generated by the Association loader (assocload)
    • AssocDiscrepancy.rpt
    • TargetDiscrepancy.rpt

  • UniProt Parser + Gene Ontology Input Files generates these files that are used as input into the Annotation loader (annotload)
    • go_ec_annot.txt - Marker-to-GO/EC annotations, J:72245
    • go_ip_annot.txt - Marker-to-GO/InterPro annotations, J:72247
    • go_spkw_annot.txt - Marker-to-GO/SwissProt keyword annotations, J:60000
    • marker_ip_annot.txt - Marker-to-InterPro annotations, J:53168

  • Output Logs
    • uniprotload.cur.log - Curator's file
    • uniprotload.proc.log - Proc file
    • uniprotload.diag.log - Diagnostic file

  • Archive - Data files from previous loads.
